# Skill: CapStation QA Checklist

Use this skill before considering any implementation complete.

## Technical QA

- [ ] Laravel app runs without errors.
- [ ] Vite assets compile.
- [ ] No missing imports.
- [ ] Routes are named.
- [ ] Locale switching works.
- [ ] Arabic is default.
- [ ] English pages work.
- [ ] No hardcoded text that should be translated.

## Design QA

- [ ] Logo is clear.
- [ ] Colors match CapStation palette.
- [ ] Website feels premium, not generic.
- [ ] Hero is impressive.
- [ ] CTA is visible.
- [ ] Cards are aligned.
- [ ] Footer is complete.
- [ ] Mobile layout is clean.

## RTL/LTR QA

- [ ] Arabic direction is RTL.
- [ ] English direction is LTR.
- [ ] Navbar works in both directions.
- [ ] Offcanvas/collapse works in both directions.
- [ ] Icons/arrows are correct.
- [ ] No horizontal scroll in Arabic.

## Animation QA

- [ ] Hero car/route animation works.
- [ ] ScrollTrigger does not break layout.
- [ ] Animations are disabled/simplified with reduced motion.
- [ ] Mobile animation is lightweight.
- [ ] No janky motion.

## SEO QA

- [ ] Title exists.
- [ ] Meta description exists.
- [ ] OG tags exist.
- [ ] Canonical exists.
- [ ] JSON-LD exists.
- [ ] H1/H2 hierarchy is correct.

## Accessibility QA

- [ ] Buttons and links are keyboard accessible.
- [ ] Icon buttons have aria-labels.
- [ ] Color contrast is good.
- [ ] FAQ accordion is accessible.
- [ ] Reduced motion is respected.
